# 业务机面单监控服务

用途：业务机不运行 ai-agent 时，也能参与订单系统的面单监控批次。

## 使用方式

1. 把整个采集客户端发布文件夹放到业务机。
2. 修改 `business_waybill_service.json`：

```json
{
  "order_server_url": "http://订单系统服务器IP:8000",
  "poll_interval_seconds": 2,
  "machine_name": "",
  "machine_label": "业务机A"
}
```

3. 双击 `启动业务机面单监控服务.bat`。
4. 订单系统网页会显示这台业务机在线。
5. 任意业务网页点击 `开始监听` 后，本服务记录本机打印组件基准。
6. 任意业务网页点击 `结束监听并加入待处理` 后，本服务上传本次打印信息原文。

## 可选开机自启

需要长期使用时，右键管理员运行：

- `安装业务机面单监控服务-开机自启.bat`

取消自启：

- `卸载业务机面单监控服务-开机自启.bat`

查看运行日志：

- `查看业务机面单监控服务日志.bat`

注意：服务主程序文件名是 `business_waybill_service.exe`，启动脚本固定调用这个英文文件名，避免 Windows `cmd` 在中文路径/中文 EXE 名下解析失败。

## 数据来源

服务只读取本机打印组件数据，不修改打印组件：

- `C:\Program Files (x86)\CNPrintTool\resources\print.db`
- `C:\Program Files (x86)\CloudPrintClient\resources\print.db`
- `C:\Program Files (x86)\CNPrintTool\log`

读取时会先复制数据库到服务目录下的 `data\waybill-monitor\component-db-copies`。

## 和 ai-agent 的关系

- 有 ai-agent：优先用 ai-agent 插件统一部署和常驻。
- 没有 ai-agent：使用本独立服务包。
- 两者不要在同一台业务机同时启用，避免同一台机器重复回传。
