订单整理工具 - 本次修复说明

一、EXE 名称已调整
1. web服务控制台.exe
2. 一键整理订单.exe
3. 订单整理管理系统.exe

二、核心逻辑调整
1. 订单整理管理系统：只负责维护导入模板、分类规则、档口规则、图片关系。
2. 一键整理订单：新增“导入模板”下拉框，整理时由前端选择模板。
3. Web 页面：继续支持模板选择，并修复模板变量读取顺序问题。

三、已修复问题
1. 修复订单整理管理系统启动时报 active_template 缺失的问题。
2. 修复模板双击加载时 name 未定义导致潜在异常的问题。
3. 修复后端管理端“当前模板”控制逻辑，避免与前端模板选择冲突。
4. 修复编译脚本中的 EXE 输出名称。
5. 加强 bat 中 python 命令引用，降低路径/变量被拆分执行的概率。

四、使用流程
1. 先打开“订单整理管理系统.exe”配置导入模板和规则。
2. 再打开“一键整理订单.exe”，选择订单 Excel、导入模板、输出方式后生成。
3. 需要网页方式时，打开“web服务控制台.exe”启动服务。
