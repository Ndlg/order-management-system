订单整理工具 V6 优化说明

一、保留旧数据可用
- 继续读取 data/system_data.enc。
- 原有模板、分类规则、档口规则、图片关系继续兼容。
- 图片关系同时支持旧 image_base64 和新 image_file。

二、图片匹配优化
- 优先使用“分类|规格”精确索引。
- 精确找不到时，在同分类内遍历匹配。
- 同一分类/规格的匹配结果会缓存，避免重复搜索。
- 支持后续把图片从 base64 迁移为 data/images 下的图片文件。

三、旧数据整理功能
位置：
订单整理管理系统.exe → 系统维护 → 整理/修复旧数据

功能：
- 自动备份 data/system_data.enc 到 data/backup。
- 修复模板、分类规则、档口规则。
- 重建图片关系索引。
- 清理重复/无效图片关系。
- 将旧 base64 图片迁移到 data/images。
- 保留旧字段，确保旧数据仍可使用。

四、启动体验优化
- 订单整理管理系统启动时增加加载窗口，显示数据文件、模板、规则、档口、图片关系数量和预览。
- web服务控制台改为懒加载，窗口先打开，点击“启动服务”后才加载 Web 服务模块。

五、EXE名称
- web服务控制台.exe
- 一键整理订单.exe
- 订单整理管理系统.exe

六、注意事项
- 编译前请关闭 dist 目录里的旧 exe，否则 Windows 会占用文件导致 PermissionError。
- 如果编译脚本一闪而过，请直接使用项目内的“一键编译全部.bat”，不要复制到 Word/WPS 后再保存。
