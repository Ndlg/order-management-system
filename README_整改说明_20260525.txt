订单整理管理系统 V7.4.7 整改说明

整改时间：2026-05-25

本次调整：
1. Web 后端版本号从 V7.4.4 修正为 V7.4.7-rectified-20260525。
2. Web 服务控制台版本号同步修正为 V7.4.7-rectified-20260525。
3. 删除重复注册的 /api/version 接口，避免接口覆盖和排查混乱。
4. 修正 /api/status 重复返回 web_version 的问题。
5. /api/status 和 /api/templates 正确返回当前启用导入模板。
6. Web 页面加载模板时自动选中当前启用模板。
7. 删除页面中重复写入 Web 后端版本文本的逻辑。
8. 生成接口增加输出方式白名单校验。
9. 生成完成或失败后自动清理上传临时文件，减少 uploads 目录堆积。
10. 下载接口限制为只能下载输出目录内文件，避免通过路径参数读取系统其他文件。
11. 按档口分文档模式会自动把输出文件夹打包成 zip，客户端电脑可直接下载全部档口文档。
12. Web 页面在文件夹输出模式下显示“点击下载全部档口压缩包”，不再只依赖服务器本机的打开输出目录。
13. 重做 Web 页面为工作台式界面，增加状态卡片、模板标识、分段式输出方式和下载按钮。
14. Web 服务读取系统配置改为只读缓存，避免刷新状态/模板时反复解密并重写大型数据文件。
15. Web 控制台延后加载 uvicorn，减少控制台启动等待。
16. Excel 图片嵌入增加缓存，同一分类规格图片不会在一次生成中重复转换。

卡顿原因记录：
1. 正式 data/system_data.enc 约 205MB，旧逻辑在 Web 状态/模板读取时默认 load_data()，会解密后再重写整库。
2. 本机测试：只读加载约 1.7 秒；默认读取并重写约 8.1 秒。页面初始化有状态和模板请求，旧逻辑会明显拖慢。
3. PyInstaller 单文件版首次启动还会解包并加载 pandas/openpyxl/cryptography/FastAPI 等重库，这是首启慢的基础成本。
4. 生成带图片文档时，PIL 打开/缩放/保存图片和 openpyxl 写入图片是主要耗时；按档口分文档会对多个文件重复执行。

本次性能优化验证：
1. get_current_system 首次读取正式数据约 1.7 秒。
2. 命中缓存后重复读取约 0.0001 秒。
3. /api/version、/api/status、/api/templates 冒烟测试通过。
