# 面单解析待处理问题

## 2026-06-01 4.0 / 5.0 混淆优先级

原始打印信息里可能同时出现多个可识别商品简称，例如：

```text
【现货新款4.0 ... 跑道5.0】4.0黑白 42 1件
```

当前规则只配置 `4.0`、`5.0` 这类商品简称关键词，无法单靠规则稳定判断最终商品简称。

后续需要修改解析代码的标题模式优先级：

1. 优先识别靠近 `尺码 + 数量` 的尾部商品片段，例如 `4.0黑白 42 1件`。
2. 尾部商品片段命中后，商品简称取 `4.0`，规格取 `黑白`。
3. 标题中间的 `跑道5.0` 这类词只作为标题干扰，不应覆盖尾部商品片段。
4. 不允许把 `4.0` 翻译成“四代”，也不允许把 `5.0` 翻译成“五代”；商品简称必须按用户确认的字面规则保存。

期望输出：

```text
商品简称：4.0
规格：黑白
尺码：42
数量：1
```
