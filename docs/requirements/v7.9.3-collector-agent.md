# 订单整理系统 v7.9.3 客户机采集助手任务书

## 任务目标
- 剔除旧独立客户端 scripts/collector_client_legacy/
- 内置官方业务机采集插件 OrderCollectorAgent
- 客户机工具只负责读取本地打印组件并回传原始面单信息，不做识别、不丢任务ID
- Web 后端统一接收并展示原文
- 客户机可登录 Web、托盘运行、开机自启、日志查看

## 主要阶段

### 阶段1：剔除旧客户端
- 删除 scripts/collector_client_legacy/ 及所有 legacy 文件
- 确认构建和主系统流程不受影响

### 阶段2：内置插件实现
- 源码位置： src/plugins/collector_agent/
- 客户机功能：读取打印组件数据库、完整回传原文、轮询 Web 获取批次指令、上传、日志记录
- 界面极简：显示连接状态、批次、组件状态、最近上传、托盘运行
- 安装目录规范：
  - 程序: C:\\Program Files\\OrderSystemCollector\\
  - 数据: C:\\ProgramData\\OrderSystemCollector\\{config,logs,cache,pending_uploads}

### 阶段3：Web 接口与 UI
- 接口：/api/collector/bind、/api/collector/poll、/api/collector/upload
- Web UI: 显示业务机在线状态、最近上传记录、查看打印原文、绑定业务机、日志查看

### 阶段4：任务上传规则
- 每条 component_rowid 都必须上传一条记录
- 不做识别、筛选或模式区分
- 不丢任何 task_id
- 上传失败需进入 pending_uploads 待重试

### 阶段5：测试要求
- 单元测试：组件读取完整、Web 批次轮询、原文上传完整
- 接口测试：poll 和 upload 正常
- 回归测试：与旧客户端行为一致，系统主流程不受影响

### 阶段6：文档与版本
- 更新文档：安装指南、绑定说明、日志查看、release policy
- 版本更新时说明是否需要业务机更新采集助手
- 构建产物：versions/v7.9.3/bin/OrderCollectorAgent_v7.9.3.exe 或 zip
- 根目录无垃圾文件或 legacy 残留

## 禁止事项
- 客户机工具不得做订单整理、Excel 导入导出、鞋款规则维护、图片管理等功能
- 不可丢任务ID或 component_rowid
- 不可在客户机生成零散临时文件
- 不可保留 legacy 代码或启动脚本

## 最终交付物
- src/plugins/collector_agent/ 插件源码
- 客户机 EXE 或 zip
- Web 接口改动
- 测试脚本及测试报告
- 文档更新
