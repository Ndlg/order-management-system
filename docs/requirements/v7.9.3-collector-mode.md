# v7.9.3 采集模式与打印原文

## 目标

v7.9.3 增加 `filtered` / `full` 两种采集模式，并保留采集工具回传的真实打印原文。默认模式仍为 `filtered`，避免影响旧业务流程。

## 配置

- 统一配置模块：`src/core/collector_config.py`
- 服务端配置文件：`data/waybill-monitor/collector_settings.json`
- 配置变更日志：`data/waybill-monitor/collector_settings.log`
- Web 与采集客户端共用接口：
  - `GET /api/collector/config`
  - `PUT /api/collector/config`

合法模式：

- `filtered`：筛选采集，保持现有监听和解析流程。
- `full`：全量采集，服务端必须保存未识别打印原文。

## 原文记录

- 统一记录模块：`src/core/collector_raw_records.py`
- 持久化文件：`data/waybill-monitor/collector_raw_records.jsonl`
- Web 查看接口：
  - `GET /api/collector/raw-records`
  - `GET /api/collector/raw-records/{record_id}`

每条记录保存：

- `collection_mode`
- `raw_print_text`
- `raw_payload_json`
- `raw_hash`
- `pattern_type`
- `parse_status`
- `collector_version`
- `collector_id`
- `created_at` / `updated_at`

## Web

Web 生成页的 `监听打印机面单` 区块新增：

- 当前采集模式显示。
- `筛选采集` / `全量采集` 切换。
- 最近采集原文列表。
- 打印原文详情与复制按钮。

## 采集客户端

旧业务机采集客户端 `scripts/collector_client_legacy/business_waybill_service.py` 新增：

- 本地配置字段 `collection_mode`。
- 启动时读取当前模式。
- poll 时同步服务端模式。
- 本地配置变化后通过 `PUT /api/collector/config` 同步给服务端。
- upload 时带上 `collection_mode`、`collector_version`、`raw_print_text`、`raw_payload_json`。

## 测试

自动化测试在 `src/tests/test_collector_mode.py`，覆盖：

- 默认模式为 `filtered`。
- 非法模式拒绝。
- 接口可切换 `filtered` / `full`。
- `full` 模式保存未识别打印原文。
- 原文特殊字符和换行可持久化。
- 上传接口保存 `collection_mode`。
