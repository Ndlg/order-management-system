# 项目整改评估

## 当前结论

当前系统已经不适合继续按“传统版”和“面单监听版”两条代码线混在一起维护。正确边界应改成：

1. 订单系统主线：五要素标准化、鞋款分类、档口、图片、输出。
2. 面单原文解析：把 `打印信息` 原文解析为五要素，可由用户维护规则。
3. Web 监听控制：只负责批次控制、业务机连接、原文/识别文件生成。
4. 采集客户端：只采集业务机打印原文；当前不进入发布包，源码单独归档。

## 已完成第一轮整改

- 新增 `app_info.py`，统一版本号、版本形态和窗口标题。
- `app.py`、`qt_admin.py`、`qt_client.py`、`qt_web_console.py`、`build_qt_windows.py` 不再各自硬编码版本号。
- Qt 构建输出从 `source/dist_qt_*` 改到版本目录下的 `build/dist_qt_*`。
- 旧业务机采集客户端源码移入 `collector_client_legacy/`，不再混在主源码根目录。
- `README.md` 和 `CHANGELOG.md` 更新为当前边界。

## 已完成重点拆分

- `waybill_files.py`：服务端文件路径、原文 Excel、批次导出、记录去重。
- `waybill_collector_reader.py`：业务机打印组件数据库读取、组件状态、采集记录。
- `waybill_monitor.py`：只保留兼容转发，不再承载新职责。

调用边界：

- `app.py` 只使用 `waybill_files.py`。
- `collector_client_legacy/business_waybill_service.py` 使用 `waybill_collector_reader.py` 采集，并复用 `waybill_files.record_key()` 去重。
- 主系统 Qt 打包不再强制包含 `waybill_monitor.py` 和 `waybill_collector_reader.py`。

## 第二轮建议

1. 把 Web 面单监听状态从 `app.py` 拆到 `waybill_server_state.py`。
2. 把 `qt_admin.py` 里的页面拆分为独立模块，优先拆 `面单解析` 页面。
3. 把数据兼容迁移逻辑从 `order_secure_common.py` 拆到单独 migration 模块。
4. 补一组面单解析回归样本，避免每次规则/代码修改后靠肉眼验证。
