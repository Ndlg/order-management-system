# 原始面单采集服务

这个工具只采集打印组件数据库里的原始 `task.msg`，不解析 JSON、不提取 `printXML`、不整理商品字段。

流程：

1. 系统端开始监听时，本工具记录每个组件数据库当前最大 `rowid`。
2. 系统端停止监听时，本工具只上传 `rowid` 大于开始值的新记录。
3. 上传内容的 `打印信息` 字段就是组件数据库里的原始 `msg` 字符串。

配置文件：`business_waybill_raw_msg_service.json`

服务端地址改这里：

```json
{
  "order_server_url": "http://127.0.0.1:8000",
  "poll_interval_seconds": 2,
  "machine_name": "",
  "machine_label": ""
}
```

注意：这个工具和旧业务机工具互不替换。不要同时运行两套工具采同一批，避免服务端收到重复业务机上传。
